Plain English — what it does

AI coding agents are confident, and confidence isn't correctness. An agent says "done" when it means "I wrote code that looks right" — a different claim from "I proved it runs," different again from "I proved it's correct." This stack closes that gap: every change is exercised the way a real user hits it before anyone calls it finished, and the call to make a release real stays with a human. It installs all 19 entrypoints together: thirteen namespaced standalone Codex capabilities plus six backward-compatible names. The active Codex hook layer injects the proof reflex into every session and subagent, routes each work prompt to its matching protocol, and continues a Stop when the latest runnable edit has not been checked or its evidence receipt is missing.

Technical architecture

Two altitudes, one rule: nothing advances on a claim, only on evidence. Every unit of work — a fix, a feature, a refactor — walks a five-gate loop. Once per version, a heavier release gate runs over everything that merged, and ends at a human's go/no-go on the tag. Typed evidence artifacts carry run, commit, claim, finding, coverage, and handoff identity between stages. A continuity bookend keeps decisions from being lost across days and machines.

Evidence moves forward without losing identity

Every standalone stage can emit or consume the same evidence contract. Visitor Audit owns the public path through the exact download; Walkthrough verifies that artifact’s version and hash before installing it in a clean environment. Downstream stages may add evidence, but cannot rewrite upstream findings or coverage.

1 · PLANTrace the real code, write the test list and done-criteria, and classify blast radius — a one-line auth change is small in lines and large in blast, and gets the deep treatment.
2 · BUILD — $dev-rigor-stack-buildTest-first. Write the failing test and watch it fail before making it pass; that, not a coverage number, is what proves the test is real.
3 · VERIFY — $dev-rigor-stack-proof-gateAdversarially try to break the claim. It survives only if it can't be refuted — killing checks that can't fail and "done" that outruns the evidence.
4 · REVIEWAudit sized to the change, plus exhaustive Walkthrough for user-facing work and Visitor Audit for every public surface and acquisition path.
5 · MERGEGreen-path only — a reviewed PR to main. No --admin, no override, no bypassing branch protection. One bad red merge undoes every green one after it.
Release gateOver the whole version: the full gauntlet driven to zero findings, the docs checked for overclaiming, a rollback plan named — then the owner's go/no-go on the tag.

Active Codex enforcement

The hook layer is active architecture, not decorative guidance. The reflex loads the never-shrink rules and evidence receipt into the coordinator and every subagent. The router injects only the task protocol that matches the prompt. The grounding ledger records runnable edits and actual executions or renders. At Stop and SubagentStop, the gate continues Codex if no check followed the latest runnable edit or the evidence receipt is absent. Codex requires the user to review and trust non-managed hooks through /hooks; installed-but-untrusted hooks are not described as enforced.

Its core idea is proportion — spend exactly the rigor a change earns. Climb the ladder only as far as the blast radius demands, and stop at the first rung that covers it. No ceremony on a one-liner; no shortcuts on the auth path.

Some rules never shrink, no matter how small the change: watch a test fail before trusting it, never merge red or with --admin, never review your own work, never claim beyond the check you actually ran, and owner decisions (tag, publish, deploy, spend, delete) stay the owner's.

Get started

For Codex Desktop:

git clone https://github.com/scottconverse/codex-dev-rigor-stack
cd codex-dev-rigor-stack
./install.sh                                             # macOS / Linux / Git Bash
powershell -ExecutionPolicy Bypass -File .\install.ps1   # Windows

Requires Git and Node.js. The active hooks use Node built-ins only. On Windows the -ExecutionPolicy Bypass prefix avoids the default "scripts are disabled on this system" block.

Installs nineteen skill entrypoints plus the active hook runtime, preserves foreign hook definitions, and merges six owned events into ~/.codex/hooks.json. Open /hooks, review and trust them, then restart Codex.

Inside a Codex session (no terminal): ask the agent to install it from the repository; it copies the skills into the configured Codex home.

For other agents (ChatGPT, Gemini, or a plain system prompt): run ./export/export-portable.sh and paste the generated portable-bundle.md into its system prompt. The export is derived from the complete canonical skills; canonical capabilities are never removed merely to make the bundle shorter.

Honest scope

What it is

A repeatable discipline that makes an agent prove its work at the layer of the claim. Model-agnostic and tool-agnostic. A way to keep long, multi-machine efforts from losing their decisions.

What it isn't

Not magic, and not a guarantee of bug-free code. It's deliberate overhead that only pays off when being wrong is expensive — so it tells you, in every gate, when to skip it.
